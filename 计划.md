# analysis 挂载通义听悟后处理（录音 + OSS + 听悟 + 钉钉）

## Summary
- 范围只覆盖 `analysis` 与其 `auto-analysis` 调用链；`mic-listen` 本轮不改。
- 新能力按显式开关启用，挂在 `analysis` 内；课程结束后由独立子进程异步执行，不阻塞 `analysis/auto-analysis` 主流程。
- 实现链路：课程音频本地整段 MP3 录制 -> 上传 OSS（分片上传）-> 调用听悟任务并轮询 -> 拉取 JSON 结果 -> 固定结构 Markdown（缺失字段填 N/A）-> 钉钉“提交通知 + 最终汇总（可多条分片）”。
- 已确认语义：同一课程时段若 analysis 重启，按“每个进程独立产出”处理，不做跨进程合并。

## Public Interfaces / Config Changes
- 新增 `analysis` 参数（并可由 `auto-analysis.analysis_args` 透传）：
  - `--tingwu-enabled`
  - `--tingwu-poll-interval-sec`（默认 30）
  - `--tingwu-max-wait-hours`（默认 6）
- 新增独立子命令：`tingwu-process --job-file <path>`（用于 analysis 异步拉起，也可手工重跑）。
- `.account` 新增且仅支持标准键名：
  - `ALIBABA_CLOUD_ACCESS_KEY_ID`
  - `ALIBABA_CLOUD_ACCESS_KEY_SECRET`
  - `TINGWU_APP_KEY`
  - `TINGWU_OSS_BUCKET`
  - `TINGWU_OSS_REGION`
  - `TINGWU_OSS_ENDPOINT`
- 依赖新增到 `requirements.txt`：官方 OSS SDK + 阿里云 OpenAPI SDK（按你确认的“接受官方 SDK”方案）。

## Implementation Changes
- 在 [parser.py](/home/placitudo/Fuck-ZJU/src/cli/parser.py) 增加上述 CLI 参数与 `tingwu-process` 子命令；在 [main.py](/home/placitudo/Fuck-ZJU/src/main.py) 接入新命令分发。
- 在 [analysis.py](/home/placitudo/Fuck-ZJU/src/live/analysis.py) 增加听悟主控：
  - `--tingwu-enabled` 时，启动前执行强校验（缺凭据/录制不可用即失败）。
  - 课程运行期间启动“纯音频录制器”（MP3，最佳努力，不补静音）。
  - 退出时落 `job.json`，并 `start_new_session=True` 拉起 `tingwu-process`，父进程退出后子进程继续跑到完成。
  - 若子进程拉起失败，立即钉钉失败告警。
- 新增纯音频录制子系统（不复用现有视频录制器逻辑）：
  - 仅抓 teacher 音频流，断流时分段，结束后合并为整段 MP3。
  - 会话目录内生成录制报告（段列表、时长、失败原因）。
- 新增听悟后处理子进程模块（`tingwu-process`）：
  - 读取 `job.json` 与 `.account`。
  - OSS 上传采用分片上传；对象路径按 `tingwu/YYYY/MM/DD/<session_folder>/...`。
  - 生成 24h 私有签名 URL（不在钉钉中暴露完整 URL）。
  - 调用听悟 CreateTask + GetTaskInfo 轮询（30s/6h，有限重试）。
  - 拉取能力 JSON（转写/章节/要点/摘要等）落到会话目录。
  - 渲染固定结构 Markdown（缺失填 N/A；“说话人细节仅保存在原始 JSON，不进 Markdown”）。
  - 钉钉发送策略：提交成功通知 1 条 + 最终汇总按长度自动拆分多条（不设上限）。
  - 任一失败路径都发送钉钉失败告警，并把错误细节落盘。
- 在 [auto_analysis.py](/home/placitudo/Fuck-ZJU/src/live/auto_analysis.py) 增加听悟启动预检（仅当 `analysis_args` 开启）：
  - 本地预检：ffmpeg、必需 `.account` 字段。
  - 远端预检：听悟鉴权探测 + OSS 端到端探针（上传微小探针对象并验证签名 URL 可读，完成后立即删除）。
  - 预检失败则 `auto-analysis` 启动即失败（符合你要求）。

## Test Plan
- CLI/配置测试：
  - `analysis` 新参数默认值与自定义值。
  - `tingwu-process` 子命令解析。
  - `auto-analysis.analysis_args` 对新增参数透传与校验。
- analysis 流程测试：
  - `--tingwu-enabled` 下缺凭据/ffmpeg 不可用时启动失败。
  - 正常退出时生成 `job.json` 并异步拉起子进程（不阻塞退出）。
  - 子进程拉起失败时失败告警触发。
- 录制器测试：
  - 断流分段、最终合并 MP3、报告落盘。
- worker 流程级 mock 测试：
  - OSS 分片上传成功路径。
  - CreateTask + 轮询完成路径，结果 JSON 下载与 Markdown 渲染。
  - 长文本分片发送（多条钉钉）与失败告警路径。
  - 轮询超时、上传失败、结果 URL 拉取失败等异常路径。
- auto-analysis 预检测试：
  - 听悟开启时预检失败即整体失败。
  - 预检探针对象会被删除。

## Assumptions
- 你会把 `.account` 调整为标准键名并补齐 `TINGWU_APP_KEY` 与 `TINGWU_OSS_*`。
- 听悟 endpoint 固定 `cn-beijing`。
- 本地 MP3 与 OSS 对象默认长期保留。
- 健康/profile 仅透传基础元信息（课程、时长、文件大小、TaskId、处理耗时、状态）。
